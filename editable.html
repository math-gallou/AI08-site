Grocery List

﻿

Une application mobile pour vos courses

Grocery List est une application Android conçue pour vous faire gagner du temps lors de la planification de repas. Combien de fois avez-vous pu vous demander "Que vais-je bien pouvoir manger cette semaine ?", certainement beaucoup de fois. Avec une base de données bien fournie, vous trouverez de quoi manger toute la semaine. Votre recette préférée n'est pas dans la liste ? Pas de soucis, vous pouvez ajouter vos propres recettes très facilement via les interfaces dédiées.
Un système de tri avancé vous aidera à retrouver efficacement vos recettes favorites.

Études utilisateurs

Pour notre projet Grocery List, nous avons prévu de cibler une population d'étudiants, ou de personnes disposant de peu de temps pour planifier leurs courses, tels des parents qui n’auraient que peu de temps libre. Nous avons donc effectué une étude auprès de nos connaissances afin de savoir ce qu’ils pensaient des applications génératrices de liste de courses. Le groupe de personnes sondés est composé de 17 étudiants et de 6 salariés.

Les premiers résultats de l'étude montrent qu'il y a clairement un marché à prendre : 75% des sondés n'utilisent pas d'application pour les aider à faire leurs courses. Par conséquent, ce sont des utilisateurs à convaincre d'utiliser l'application. Il faudra avoir une expérience utilisateur idéale pour garder les utilisateurs sur l'application et les convaincre de changer leurs habitudes.

Ce second graphique confirme l'importance d'avoir une expérience utilisateur optimale pour convertir les utilisateurs et les attirer sur l'application.

Étude de l'existant

Le marché des applications de liste de course ou de planning de plat n'est pas nouveau dans nos magasins d’applications. Il existe beaucoup d’applications qui complètent au moins l’une de ces deux activités, mais jamais de la façon dont nous le souhaitons. Voici une présentation des fonctionnalités que l’on retrouve le plus pour ce type d’applications (exemple : Bring, Jow) :

  • Base de données de recettes et / ou d’ingrédients existantes,
  • Personnalisation d’une recette et / ou d’un ingrédient, peu importe sa source,
  • Génération d’une liste de courses à partir d’une sélection de recettes,
  • Création manuelle ou automatique d’un planning de recette sur une base hebdomadaire ou mensuelle,
  • Enregistrer une recette depuis un autre site internet,
  • Synchronisation des données via le cloud grâce à un identifiant,
  • Affichage des informations nutritionnelles,
  • Génération d’un planning de recettes selon un paramétrage,
  • Paramétrer la génération d’un menu selon un régime alimentaire, un nombre de personnes avec une distinction enfant / adulte, exclusion ou inclusion d’un ingrédient, matériel de cuisine à disposition,
  • Synchronisation des informations nutritionnelles des recettes avec les applications de suivi fitness,
  • Proposer une liste de recettes en se basant sur les ingrédients déjà en possession de l’utilisateur,
  • Ajout des prix sur les ingrédients afin d’estimer le coût d’une recette / le total de la liste de course,
  • Lien avec les grandes enseignes de supermarché pour générer automatiquement le panier et le récupérer en Drive.

Cette étude de l'existant nous permet d'identifier le point sur lequel nous serons les plus convaincants : la gratuité de notre application. Cela diffère grandement des applications entièrement payantes, ou proposant un accès complet à toutes les fonctionnalités via un abonnement, pouvant résulter en une frustration pour l’utilisateur.

Fonctionnalités

On remarque que les fonctionnalités que nous proposons sont relativement attendues par une grande partie des utilisateurs potentiels. On voit aussi qu'il est important de leur proposer des solutions car naturellement, certains utilisateurs ne savent pas réellement quelles fonctionnalités pourraient les aider.
